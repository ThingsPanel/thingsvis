# 开发：TDD 领航员

执行 TDD 流程：红灯（写失败的测试）→ 绿灯（让测试通过）→ 重构。

## 快速开始

1. 运行：`.specify/scripts/powershell/check-prerequisites.ps1 -Json -RequireTasks -IncludeTasks`
2. 加载 `tasks.md` 获取任务列表
3. 对每个任务执行 TDD 循环
4. 完成后标记任务为 `[x]`

## TDD 流程

复制并跟踪进度：

```
TDD 进度：
- [ ] 阶段 1：红灯 - 写失败的测试
- [ ] 阶段 2：绿灯 - 实现代码让测试通过
- [ ] 阶段 3：重构 - 提升代码质量
- [ ] 阶段 4：标记任务完成
```

### 🔴 阶段 1：红灯（测试先行）

**目标**：根据验收标准创建「考试题」

1. 加载需求：`specs/[feature]/spec.md`
2. 从用户故事中提取验收标准
3. 生成**会失败**的测试用例

```bash
# 运行测试 - 预期全部红灯
pnpm test
```

测试结构示例：
```typescript
describe('US1: 功能名称', () => {
  it('应该 [验收标准 1]', () => {
    // 准备
    // 执行
    // 断言 - 初始应该失败
  });
});
```

### 🟢 阶段 2：绿灯（实现代码）

**目标**：写最少的代码让测试通过

1. 加载任务：`specs/[feature]/tasks.md`
2. 实现当前任务的代码
3. 每次修改后运行测试

```bash
# 运行测试 - 看着它们变绿
pnpm test
```

**规则**：
- 只写刚好能让测试通过的代码
- 不要过早优化
- 不要超出当前任务范围

### 🔄 阶段 3：重构

**目标**：在不改变行为的前提下改进代码

1. 所有测试必须保持绿灯
2. 应用代码审查清单
3. 消除重复
4. 改进命名

## 任务执行

**tasks.md 中的任务格式**：
```markdown
- [ ] T001 创建项目结构
- [ ] T005 [P] 实现认证中间件（可并行）
- [ ] T012 [US1] 创建 User 模型（属于 US1）
```

**执行规则**：
- 顺序任务：按顺序执行
- `[P]` 任务：可以并行执行
- `[US#]` 任务：按用户故事分组

**完成每个任务后**：
```markdown
- [x] T001 创建项目结构  ← 标记完成
```

## 代码审查清单

标记任务完成前检查：

```
代码质量：
- [ ] 没有使用 `any` 类型
- [ ] 没有未处理的错误
- [ ] 没有遗留的 console.log
- [ ] 变量命名语义化
- [ ] 有 JSDoc 注释
- [ ] 测试覆盖率足够
```

## 异常处理

**如果测试意外失败**：
1. 停止实现
2. 分析失败原因
3. 修复测试或代码
4. 绿灯后才能继续

**如果任务被阻塞**：
1. 记录阻塞原因
2. 跳到下一个未阻塞的任务
3. 阻塞解除后返回

## 边界

**职责范围**：写测试、实现代码、运行测试、标记任务
**不涉及**：修改需求、修改方案、架构决策
