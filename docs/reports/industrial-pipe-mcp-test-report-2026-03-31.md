# 工业管道组件 MCP 测试报告

- 测试日期: 2026-03-31
- 测试方式: Chrome DevTools MCP 手工交互 + 运行时状态检查
- 测试地址: `http://localhost:3000/main#/editor`
- 测试对象: `工业管道` 组件

## 测试范围

1. 组件选中与属性面板联动
2. 样式属性修改后是否立即反映到画布
3. 流动效果与发光效果开关
4. 组件库点击新增行为
5. 保存状态与控制台告警

## 通过项

1. 已存在的 `工业管道` 实例可以被选中，选中后会出现起点、终点、路径中点的连接/编辑手柄。
2. `管道颜色` 从 `#2563eb` 修改为 `#ff0000` 后，SVG 主路径 `stroke` 已同步更新为 `#ff0000`。
3. 打开 `flowEnabled` 后，流动路径会显示，DOM 中第三条 `path` 出现 `stroke="#38bdf8"`、`stroke-dasharray="16 24"`，并持续更新 `strokeDashoffset`。
4. 打开 `发光效果` 后，流动路径会挂载 SVG filter，`filter="url(#pipe-glow-...)"` 生效。
5. 点击 `保存` 后，页面状态会回到“所有更改已保存”。

## 发现的问题

### P1: 组件库点击卡片无法新增组件

- 复现步骤:
  1. 清空当前选中态，使右侧切回画布设置。
  2. 在左侧组件库点击 `流量计`。
  3. 再点击 `工业管道`。
- 实际结果:
  - 画布未新增任何节点。
  - 运行时 `window.__THINGSVIS_DEV__.getState().nodesById` 数量始终为 `1`。
  - 右侧面板保持在画布设置，没有切到新组件属性。
- 影响:
  - 无法稳定从组件库新建工业组件，直接阻塞 `工业管道` 的新增测试和正常使用。
- 备注:
  - 该问题看起来不只影响 `工业管道`，`流量计` 同样未能新增，更像编辑器侧新增链路异常。

### P1: 数值输入与实际渲染结果不一致

- 复现步骤:
  1. 选中 `工业管道`。
  2. 在 `strokeWidth` 输入框中继续输入 `24`。
- 实际结果:
  - 输入框显示值变成了 `1224`。
  - 但 SVG 实际渲染被钳制，主路径 `stroke-width` 为 `80`，边框为 `120`，流动层为 `40`。
- 影响:
  - 属性面板展示值和真实渲染值不一致，用户无法判断当前生效的线宽。
  - 如果最终保存的是超限原值，会造成后续编辑和渲染结果继续偏差。

### P2: 多个属性标签未完成本地化

- 现象:
  - 右侧面板中存在直接显示原始字段名的情况，包括:
    - `strokeWidth`
    - `glowIntensity`
    - `flowEnabled`
    - `flowSpeed`
    - `flowLength`
    - `flowSpacing`
- 对比:
  - `管道颜色`、`管壁颜色`、`流动颜色`、`流动方向`、`起点节点`、`终点节点` 已显示中文。
- 影响:
  - 面板中英混排，专业用户尚可理解，但会明显降低配置可读性和产品完成度。

### P2: 控制台持续出现 `opacity` 控件配置告警

- 控制台告警:
  - `[getControls] Widget defines base-style-managed path "opacity" in controls. It will be hidden. Please remove it from the widget's controls.ts.`
- 实际影响:
  - 该字段在面板中被隐藏，但每次加载都会重复告警。
  - 会污染调试输出，也说明当前控件定义和宿主面板的职责边界不一致。

## 暂未下结论的点

1. `起点节点` / `终点节点` 下拉框当前只看到 `(无连接)`。
2. 本次测试时运行时画布里只有 1 个节点，无法直接判定这是 `nodeSelect` 本身失效，还是当前没有可连接目标导致的正常结果。
3. 如果要把这个点测实，建议先修复“组件库点击无法新增组件”，然后再复测节点绑定链路。

## 建议优先级

1. 先排查组件库新增链路，确保工业组件能稳定创建。
2. 修正数值控件的输入约束和回写逻辑，避免面板值与渲染值分叉。
3. 补齐 `工业管道` 控件的本地化文案。
4. 清理 `controls.ts` 中不该继续声明的 `opacity` 字段。
