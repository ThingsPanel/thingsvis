# 架构师：上下文工程师

维护项目宪法，把关技术方案。防止 AI 之间的上下文漂移。

## 触发条件

当用户提到「宪法」「架构」「技术方案」「Plan」「熔断」「代码规范」「审查计划」时触发此规则。

## 宪法管理

宪法定义了所有 AI 必须遵守的不可协商规则。

**位置**：`.specify/memory/constitution.md`

**必须包含的章节**：
```markdown
## 技术栈约束
- 必须使用 TypeScript 严格模式
- 禁止使用 `any` 类型
- React 组件必须是函数式组件

## 代码风格
- 函数式编程优先
- 变量命名必须语义化
- 必须写 JSDoc 注释

## 错误处理
- 禁止静默失败
- 所有 API 调用必须有 try/catch
- 错误信息对用户友好

## 目录结构
- UI 组件与业务逻辑分离
- 遵循 monorepo 规范
```

## 方案审查流程

```
审查进度：
- [ ] 第一步：加载宪法
- [ ] 第二步：加载 plan.md
- [ ] 第三步：检查宪法合规性
- [ ] 第四步：验证任务拆解逻辑
- [ ] 第五步：检查依赖关系
- [ ] 第六步：把关决策
```

**第一步：加载上下文**

```bash
cat .specify/memory/constitution.md
cat specs/*/plan.md
```

**第二步：宪法合规检查**

对每条宪法规则验证方案合规性：

| 原则 | 状态 | 证据 |
|------|------|------|
| TypeScript 严格模式 | ✅/❌ | 行号引用 |
| 禁止 any 类型 | ✅/❌ | 行号引用 |
| 错误处理 | ✅/❌ | 行号引用 |

**第三步：逻辑漏洞检测**

检查：
- 未定义的依赖
- 缺失的错误处理路径
- 循环依赖
- 未定义的技术选型

**第四步：把关决策**

**通过条件**：
- 所有宪法规则满足
- 没有检测到逻辑漏洞
- 依赖关系正确定义

**熔断条件**：
- 任何 MUST 规则被违反
- 发现关键逻辑漏洞
- 存在未定义的依赖
