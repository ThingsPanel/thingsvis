# 图层系统 - 快速启动指南

## 📋 当前问题

Editor.tsx 中的图层面板是**硬编码的 Mock 数据**：

```tsx
const [layers, setLayers] = useState<Layer[]>([
  { id: "group1", name: "页面组 1", ... },  // 假数据
  { id: "chart1", name: "折线图", ... },    // 与画布无关
]);
```

这些数据与 `kernelState.nodesById` 完全脱节，用户看到的图层与画布组件不对应。

---

## 🎯 核心目标

将图层面板从 Mock UI 升级为**真正的编辑工具**：

| 现状 | 目标 |
|------|------|
| 硬编码假数据 | 实时反映 Kernel 中的组件 |
| 点击无响应 | 双向选择联动 |
| 无法排序 | 拖拽调整 z-index |
| 眼睛/锁图标无效 | 真正控制可见性/锁定 |

---

## 🔍 行业对比结论

| 平台 | 层级模型 | 我们的选择 |
|------|----------|------------|
| **Figma** | 无限嵌套 Frame | ❌ 过于复杂 |
| **DataV** | 平铺 + 分组 | ✅ 符合大屏场景 |
| **Superset** | 无图层概念 | ❌ 功能不足 |
| **Adobe XD** | 画板 + Group | ⚠️ 中等复杂度 |

**最终方案**：**平铺 + 单层分组**（参考 DataV）

## 📐 技术架构

### 数据流

```
┌─────────────────────────────────────────────────────────────┐
│                      Kernel Store                           │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ nodesById   │  │ layerOrder  │  │ layerGroups │         │
│  │ (组件数据)  │  │ (渲染顺序)  │  │ (分组信息)  │         │
│  └─────┬───────┘  └─────┬───────┘  └─────┬───────┘         │
└────────┼────────────────┼────────────────┼─────────────────┘
         │                │                │
         ▼                ▼                ▼
┌─────────────────────────────────────────────────────────────┐
│                     LayerPanel (UI)                         │
│  - 订阅 Store 状态                                          │
│  - 渲染虚拟滚动列表                                         │
│  - 处理拖拽排序                                             │
│  - 分发 actions 到 Store                                    │
└─────────────────────────────────────────────────────────────┘
```

### 新增 Kernel Actions

```typescript
// 层级操作
reorderLayers(nodeIds: string[], targetIndex: number)
bringToFront(nodeIds: string[])
sendToBack(nodeIds: string[])

// 可见性/锁定
setNodeVisible(nodeId: string, visible: boolean)
setNodeLocked(nodeId: string, locked: boolean)

// 分组
createGroup(nodeIds: string[], name?: string): string
ungroup(groupId: string)
```

## ❓ 需要您确认的决策

1. **分组层级**：是否只支持单层分组？还是需要嵌套分组？
2. **缩略图预览**：是否需要在图层条目中显示组件缩略图？（增加复杂度）
3. **快捷键**：是否需要定义图层操作的快捷键？（如 Ctrl+[ 下移一层）
4. **图层锁定逻辑**：锁定后是否允许在图层面板中选中？（仅禁止画布操作）
