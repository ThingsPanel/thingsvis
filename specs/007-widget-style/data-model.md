# Data Model: Superset 风格优先的数据配置与绑定

**Branch**: `007-widget-style`  
**Date**: 2025-12-30  
**Spec**: [spec.md](./spec.md)

This is a conceptual model for the MVP. It documents entities, relationships, validation rules, and state transitions.

## Entities

### 1) Data Source (runtime)

- **Entity**: `DataSourceRuntimeState` (kernel runtime)
- **Location today**: `packages/thingsvis-kernel/src/store/KernelStore.ts`
- **Fields**:
  - `id: string`
  - `data: any` (snapshot payload)
  - `status: 'connected' | 'disconnected' | 'error' | 'loading'`
  - `error?: string`
  - `lastUpdated: number`
- **Notes**:
  - Field Picker reads `data` to derive selectable paths.

### 2) Node (persisted)

- **Entity**: `NodeSchemaType` (schema)
- **Location today**: `packages/thingsvis-schema/src/node-schema.ts`
- **Key fields**:
  - `props?: Record<string, any>`
  - `style?: Record<string, any>` (currently not used by `PropertyResolver` in MVP)
  - `data?: DataBinding[]` (explicit bindings that can override props)

### 3) Data Binding (persisted)

- **Entity**: `DataBinding`
- **Location today**: `packages/thingsvis-schema/src/datasource/index.ts`
- **Fields**:
  - `targetProp: string` (which prop key the binding overrides)
  - `expression: string` (MUST match `/^\{\{.*\}\}$/`)
- **Behavior**:
  - At render time, `@thingsvis/ui` evaluates `expression` against context `{ ds: kernelState.dataSources }`.
  - If the resolved value is not `null`/`undefined`, it overrides `props[targetProp]`.

### 4) Plugin entry (authoring)

- **Entity**: `PluginMainModule`
- **Location today**: `packages/thingsvis-schema/src/plugin-module.ts`
- **Key fields**:
  - `schema?: any` (typically a Zod schema for plugin props)
  - **New (MVP)**: `controls?: PluginControls` (serializable panel definition)

### 5) Controls Definition (new, persisted in plugin module)

- **Entity**: `PluginControls`
- **Purpose**: Studio uses this to generate the property panel dynamically.
- **Relationships**:
  - Plugin → has one optional `controls` object.
  - `PluginControls` → contains multiple `ControlField` entries.

#### PluginControls

- **Fields (MVP)**:
  - `groups: ControlGroup[]`

#### ControlGroup

- **Fields (MVP)**:
  - `id: 'Content' | 'Style' | 'Data' | 'Advanced'`
  - `label?: string`
  - `fields: ControlField[]`

#### ControlField

- **Fields (MVP)**:
  - `path: string` (MVP: flat prop key like `text`, `fill`, `fontSize`)
  - `label: string`
  - `kind: 'string' | 'number' | 'boolean' | 'color' | 'select' | 'json'`
  - `options?: Array<{ label: string; value: string | number }>` (when `kind='select'`)
  - `default?: unknown`
  - `binding?: { enabled: boolean; modes: Array<'static' | 'field' | 'expr' | 'rule'> }`

## Validation Rules

- `DataBinding.expression` MUST be wrapped in `{{ ... }}` (already enforced by Zod).
- For MVP, `ControlField.path` MUST be a non-empty string and should map to the plugin’s props key.
- Field-binding expressions generated by Studio SHOULD follow: `{{ ds.<id>.data.<path> }}`.

## State Transitions (per field)

A Control field can transition between:

1. **Static**
   - Persisted as `node.schemaRef.props[path] = value` and no corresponding `DataBinding` for `targetProp=path`.
2. **Field-bound**
   - Persisted as `DataBinding { targetProp: path, expression: "{{ ds.<id>.data.<fieldPath> }}" }`.
3. **Expr-bound**
   - Persisted as `DataBinding { targetProp: path, expression: "{{ ... }}" }`.

The panel determines the current mode by inspecting whether a `DataBinding` exists for `targetProp=path`, and (optionally) whether the expression matches the canonical field-binding prefix.
